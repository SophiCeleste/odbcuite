## Source Files

### `ns_utils.py`

The primary module. All extract scripts import from here. Organized into
ten named sections delimited by banner comments:

| Section | Key symbols |
|---|---|
| Logging | `log_setup` |
| Console Output | `cprint` |
| Config | `load_config` |
| NetSuite ODBC | `check_odbc_driver`, `connect_netsuite`, `sql_to_df` |
| DuckDB | `load_duckdb` |
| Azure SQL | `connect_azure_sql`, `load_azure_sql` |
| SCD Type 2 | `_scd2_diff` (private), `scd2_load_duckdb`, `scd2_load_azure` |
| Timing | `format_elapsed` |
| Plant/Manufacturer Resolution | `resolve_id_list`, `resolve_manufacturer_type`, `_DEFAULT_ATX_PLANTS`, `_DEFAULT_VENDOR_PLANTS` |
| Business Days | `business_days_diff`, `bucket_days_late` |

**Line count:** ~970 lines.

### `ns_token.py`

Standalone TBA token generator. Depends on `ns_utils.load_config`. Loads
`config.json` and extracts prod credentials at import time (module-level side
effect). Exports one public function: `build_token_password()`.

**Line count:** ~86 lines.

## Configuration Files

### `pyproject.toml`

Declares the package as `odbcuite` v0.1.0, requires Python 3.9+, lists six
runtime dependencies (`pyodbc`, `duckdb`, `pandas`, `numpy`, `sqlalchemy`,
`azure-identity`), and registers the two source modules for the package
(`py-modules = ["ns_utils", "ns_token"]`).

### `config.json` (not committed)

Runtime credential and environment config. Required at runtime; absent from
the repository. Keys used by the library:

| Key path | Used by |
|---|---|
| `env` | `connect_netsuite`, callers for database routing |
| `netsuite.dev.dsn`, `.uid`, `.pwd` | `connect_netsuite` (dev) |
| `netsuite.prod.dsn`, `.account_id`, `.consumer_key`, `.consumer_secret`, `.token_id`, `.token_secret` | `ns_token.py`, `connect_netsuite` (prod) |
| `databases.dev.path` | `load_duckdb`, `scd2_load_duckdb` |
| `databases.prod.server`, `.database`, `.odbc_driver`, `.uid`?, `.pwd`? | `connect_azure_sql`, `load_azure_sql`, `scd2_load_azure` |
| `tables.*` | caller scripts (fully qualified table names) |
| `raw_folder` | caller scripts (CSV export directory) |

## Runtime-Generated Directories

### `logs/`

Created by `log_setup()` on first script run. Structure:

```
logs/
в””в”Ђв”Ђ <script_stem>/
    в”њв”Ђв”Ђ <script_stem>_20260615_083012.log
    в””в”Ђв”Ђ <script_stem>_20260616_083045.log
```

One subdirectory per calling script name; one log file per run, timestamped
to the second. Not committed to git.

### `raw/`

CSV export staging area referenced by `config["raw_folder"]`. Not managed by
the library вЂ” caller scripts write here. Not committed to git.

## Generated / Build Artifacts

### `odbcuite.egg-info/`

Created by `pip install -e .`. Safe to delete and regenerate. Contains
package metadata derived from `pyproject.toml`.

### `__pycache__/`

Standard Python bytecode cache. Generated automatically; not committed.

## Planning Documents

`.planning/codebase/` contains analysis documents describing the codebase as
it exists. These are reference documents for planning and development work,
not build or runtime artifacts.

| File | Contents |
|---|---|
| `ARCHITECTURE.md` | System design, data flow, auth architecture, key design decisions |
| `STRUCTURE.md` | Repository layout, file inventory, config keys |
| `STACK.md` | Language, runtime, package manager, all dependencies |
| `INTEGRATIONS.md` | External services, auth flows, storage targets, observability |
| `CONVENTIONS.md` | Naming patterns, code style, docstring format, error handling, logging |

## What Is Not Present

- **Tests:** No test files, no `tests/` directory, no pytest or unittest
  configuration.
- **CI/CD:** No GitHub Actions workflows, no Azure Pipelines, no Dockerfile.
- **Dependency lockfile:** No `requirements.txt` with pinned versions.
- **Linter/formatter config:** No `.flake8`, `.pylintrc`, or `[tool.ruff]`
  section in `pyproject.toml`.
- **Multiple packages or subpackages:** Both source files are flat modules
  registered via `py-modules`; no `src/` layout, no `__init__.py`.
